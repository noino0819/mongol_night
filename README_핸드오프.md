# 초원의 밤 v2 — 클로드코드 핸드오프 패키지

## 📦 구성
```
steppe-night/
├─ CLAUDE.md                     ← 클로드코드가 자동으로 읽는 프로젝트 규칙 (건드리지 말 것)
├─ README_핸드오프.md            ← 이 파일 (사람용)
├─ app/
│  └─ 초원의밤_몽골게임팩.html   ← v1 구현체 (15게임 동작). index.html로 rename해서 시작
├─ docs/
│  ├─ 초원의밤_v2_스케일업기획서.md
│  ├─ 초원의밤_v2_디자인UX기획서.md
│  ├─ 초원의밤_확장기획서.md
│  ├─ 밸런스게임_기획서.md
│  └─ 밸런스게임_추가800.md
└─ reference/                    ← ★ 여기에 도트 레퍼런스 이미지를 넣으세요
   └─ README.md
```

## 🚀 사용법
1. 이 폴더를 통째로 새 깃 리포에 풀기 (리포명 추천: `steppe-night`)
2. `reference/`에 마음에 드는 도트게임 스크린샷/무드 이미지 넣기 (클로드가 스타일 참고)
3. 리포에서 클로드코드 실행 → 아래 시작 프롬프트 복붙

## 📋 시작 프롬프트 (복붙용)

```
CLAUDE.md와 docs/ 기획서 5종을 읽고 프로젝트를 파악해줘.
reference/ 폴더의 이미지들은 도트 아트 스타일 레퍼런스야.

작업은 스케일업 기획서 6장 마일스톤 순서대로:
M1(PWA화 + 도트 디자인 킷)부터 시작하고, 각 마일스톤 완료 시
CLAUDE.md의 검증 스크립트를 돌리고 결과를 보고한 뒤 다음으로 넘어가줘.
독립적인 작업(스프라이트 제작, 문항 데이터, 스토리 저작)은
서브에이전트로 병렬 처리하되 확장기획서 C장 충돌 방지 규칙을 지켜줘.

첫 단계: app/초원의밤_몽골게임팩.html을 index.html로 옮기고
현재 상태에서 검증 스크립트가 통과하는지부터 확인해줘.
```

## ⚠️ 사람이 직접 해야 하는 것 (클로드가 못 하는 것)
- GitHub Pages 활성화 (리포 Settings → Pages → main 브랜치)
- 실기기 테스트: 배포 URL을 본인 폰(iOS+Android 각 1대)에서 열어 PWA 설치 → 비행기 모드 → 전 게임 확인
- OG 미리보기 확인: 배포 URL을 카톡 나에게 보내기로 공유해서 이미지·제목 확인
- 출국 전 일행 전원 설치 (연결 가이드 페이지의 QR 활용)

## 🗓️ 진행 팁
- 마일스톤 하나 끝날 때마다 커밋시키고 브라우저로 열어 눈으로 확인 → 이상하면 그 자리에서 피드백 (몰아서 보면 되돌리기 힘듦)
- 스토리("고비의 별")와 상식퀴즈 문항은 결과물 검수가 특히 중요 — 문항 오답/스토리 막다른 길은 클로드에게 "그래프 검증 스크립트 결과 보여줘"로 증빙 요구
- 막히면 해당 마일스톤 + 에러 로그를 들고 원래 채팅으로 컴백 ㅋㅋ
