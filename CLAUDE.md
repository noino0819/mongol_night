## 핵심 규칙 (전 에이전트 공통)
1. **오프라인 100%**: 외부 CDN/폰트/이미지 요청 0건. 에셋은 리포 내 정적 파일 or base64 인라인만.
2. **게임은 완전 자립 — index.html·sw.js·core.js 무수정이 원칙.** 모든 게임(20+종)이 `games/X.js` 하나에 마크업·CSS·스프라이트·로직·리셋을 담는다. 각 파일 상단에서 `snAddScreen(name, html)`(화면)·`snAddCss(css)`(전용 CSS)·`snAddSprites({...})`(전용 스프라이트)를 등록하고, 끝에서 `snRegisterGame(name, resetFn)`(리셋)을 등록. index.html엔 셸 화면(home/settings/mp/kitchen)+공통 CSS만, 공용 스프라이트는 `assets/sprites.js`, 폰트는 `assets/font.css`.
   **새 게임 추가 = `games/X.js` 신설 + `games/manifest.js` 로드목록 1줄 + `shell.js` 카탈로그 1줄** (이 3곳이 유일한 접점). 기존 게임 수정은 그 게임 파일 하나만 건드림 → 세션 충돌 없음. 표본: 아무 `games/*.js`의 상단 `snAddScreen` 블록.
3. 게임별 ID/함수 접두어 고정 (확장기획서 A-3장 표 준수). 신규 예약: lv, bz, nb, gq(상식퀴즈), ta(텍스트 어드벤처), guide(연결 가이드).
4. 색상은 CSS 변수만 (스케일업 기획서 2-2 팔레트). 스프라이트는 16×16 문자열 맵 → canvas 렌더 (이미지 파일 금지, 아이콘/OG 등 앱 셸 에셋만 예외). **신규 스프라이트는 `docs/도트스프라이트_가이드.md`의 법칙 필수 준수** — 의미가 겹치면 재사용, 독립 개념이면 신규 제작(다양성 우선). **게임 전용 스프라이트는 index.html SPR이 아니라 그 게임의 games/*.js에서 `snAddSprites()`로 등록** (병렬 세션 충돌 방지 — 가이드 1장). 제작 후 `node tools/sprite-lint.mjs`.
5. 사용자 입력 → innerHTML 경로는 `escHtml()` 필수. 타이머는 상태 객체에 저장하고 Reset 함수에서 clear.
6. 픽셀 폰트(Galmuri 등)는 OFL 라이선스 고지를 설정 화면에 포함 (법적 필수).
7. 카피 톤: 가벼운 반말. **ㅋㅋ·ㅎㅎ 등 자모 웃음 표기 금지** (2026-07 유저 지시로 전체 제거됨 — 다시 넣지 말 것). 버튼은 동사 라벨. 텡그리(GM)만 신비 츤데레체.

## 작업 순서 (마일스톤 — 스케일업 기획서 6장)
M1 PWA화 + 도트 디자인 킷 → M2 홈 IA → M3 게임 콘텐츠(이월 3종·상식퀴즈·밸런스 통합·스프라이트 20종) → M4 텍스트 어드벤처 → M5 앱 셸 에셋·스킨 일괄 적용·QA
**M1 완료 전에 게임 스킨 작업 금지** (로직 선행은 허용).

## 검증 (매 작업 후 필수)
```bash
node -e "
const fs=require('fs');
const html=fs.readFileSync('index.html','utf8');
const m=html.match(/<script>([\s\S]*)<\/script>/);
new Function(m[1]); console.log('JS OK', m[1].length);
"
```
- 게임 판정/정산 로직은 node로 추출해 단위 케이스 3개 이상 통과시킬 것.
- 릴리스 전: 스케일업 기획서 7장 DoD + 디자인 기획서 9장 DoD 전 항목 체크.
