## 1. PWA 아키텍처

### 1-1. 배포 구조
- 호스팅: **GitHub Pages** (무료 HTTPS). 리포 예: `steppe-night` → `https://{user}.github.io/steppe-night/`
- 사용 흐름: 출국 전 와이파이에서 각자 접속 → 설치 → 이후 인터넷 0%에서 앱 아이콘으로 실행.

### 1-2. 파일 구조 (이 이상 쪼개지 말 것)
```
/index.html          ← 앱 전체 (기존 게임팩 확장. HTML+CSS+JS 계속 인라인 유지)
/manifest.webmanifest
/sw.js               ← 서비스워커
/icons/icon-192.png  ← 도트 양 아이콘 (아래 6장 캐릭터 '보르')
/icons/icon-512.png
/icons/maskable-512.png
```

### 1-3. manifest 스펙
```json
{
  "name": "초원의 밤 — 몽골 파티게임",
  "short_name": "초원의밤",
  "start_url": "./",
  "display": "standalone",
  "orientation": "portrait",
  "background_color": "#0E1526",
  "theme_color": "#0E1526",
  "icons": [ {192}, {512}, {maskable 512} ]
}
```

### 1-4. 서비스워커 전략
- 설치 시 **전체 precache** (파일 목록 + 버전 상수 `const V = "v2.0.0"` — 릴리스마다 수동 bump).
- fetch: cache-first (오프라인 절대 보장). 
- 업데이트: 새 SW 감지 시 앱 내 토스트 "🆕 새 버전! 탭해서 새로고침" → skipWaiting + reload.
- **오프라인 준비 배지**: 홈 상단에 SW 설치 완료 여부 표시. `✅ 오프라인 준비 완료 — 비행기 모드에서도 됩니다` / `⏳ 다운로드 중... 와이파이 유지`. 이 배지가 v2 신뢰도의 핵심 UX.

### 1-5. 설치 유도
- Android/Chrome: `beforeinstallprompt` 캡처 → 홈에 [📲 앱으로 설치] 버튼 노출.
- iOS/Safari: 자동 설치 불가 → 안내 모달: "공유 버튼 → 홈 화면에 추가" 3단계 도트 일러스트로 안내.
- 이미 standalone 실행 중(`display-mode: standalone`)이면 설치 UI 전부 숨김.

### 1-6. 수용 기준
- [ ] 최초 방문 → 비행기 모드 → 재실행 시 모든 게임 정상 동작
- [ ] Lighthouse PWA 체크 통과 (installable)
- [ ] 버전 bump 후 재접속 시 업데이트 토스트 동작

## 2. 도트 디자인 시스템 (전면 리뉴얼)

### 2-1. 아트 디렉션
- 컨셉: **"밤의 초원 8비트 RPG"** — 레트로 게임보이/JRPG 감성 + 몽골 밤하늘.
- 픽셀 그리드: UI 요소는 4px 배수. 곡선·그라데이션 금지 → 계단식 픽셀 표현.
- `image-rendering: pixelated` 전역 적용 (canvas·스프라이트).

### 2-2. 팔레트 (기존 토큰 유지 + 도트용 8색 추가)
```
--px-black:#1A1423  --px-navy:#2A2440  --px-sand:#E8C170
--px-grass:#5FA052  --px-sky:#6FB8D6   --px-red:#D95763
--px-white:#F4EEE0  --px-brown:#8A6044
```
스프라이트는 이 8색 + 투명만 사용 (일관성 강제).

### 2-3. 픽셀 폰트
- **Galmuri11** 또는 **둥근모꼴+Fixedsys(DungGeunMo)** — 둘 다 무료 라이선스(SIL OFL / 공개 배포) 한글 픽셀 폰트.
- woff2를 base64로 index.html에 인라인 임베드 (외부 요청 금지 원칙 유지). 용량 확인 후 서브셋(KS X 1001 한글 2350자 + 영숫자) 필수 — 목표 300KB 이하.
- 적용: 제목·버튼·숫자 = 픽셀 폰트 / 본문 긴 문장(기획서·설명)은 시스템 폰트 유지 (가독성).

### 2-4. UI 컴포넌트 픽셀화 규칙
- 버튼/카드: border-radius 제거 → **픽셀 프레임** (box-shadow 계단 4px: `box-shadow: 0 4px 0 #000, inset 0 4px 0 rgba(255,255,255,.15)` 스타일). 눌림 시 translateY(4px)+그림자 제거.
- 팝업(lifeModal/mbAsk): RPG 대화창 스타일 — 두꺼운 이중 테두리 + 코너 픽셀 장식.
- 프로그레스/타이머: 픽셀 하트·양 아이콘 게이지.
- 별하늘 배경: 기존 트윙클 유지하되 별을 2×2/4×4 사각 픽셀로.
- 사운드(선택): Web Audio 합성 8비트 비프 3종(선택/정답/폭발). 설정에서 on/off, 기본 off.

### 2-5. 스프라이트 시스템 (캐릭터 구현 방식)
- 포맷: **16×16 문자열 맵** → JS 렌더러가 canvas에 그림. 이미지 파일 불필요, diff 관리 쉬움.
```js
// '.'=투명, 나머지는 팔레트 인덱스(0~7)
const SPR = { bor: [
  "....7777........",
  "...766667.......",  // ...16줄
]};
function drawSprite(cv, key, scale=4){ /* fillRect 렌더 */ }
```
- 사용처: 홈 카테고리 아이콘, 게임 헤더(마스코트 + 이름), 승리 화면(마스코트 점프 2프레임 애니), TRPG GM 초상.
- 애니메이션: 2프레임 idle (600ms 토글)이면 충분. 과욕 금지.

## 3. 홈 IA 개편 (게임 뎁스)

### 3-1. 구조 (2뎁스)
```
홈
├─ [오프라인 준비 배지] [📲 설치] [⚙️ 설정] [📡 연결 가이드]
├─ 🔍 인원 필터 칩: [전체] [2인] [3-4인] [5인+]   ← 탭 시 카테고리 내 게임 필터
├─ ⏱️ 최근 한 판 (마지막 플레이 1개, 바로가기)  ← localStorage
├─ ⭐ 즐겨찾기 (카드 길게 눌러 등록/해제)
└─ 카테고리 폴더 6개 (탭하면 펼침/접힘, 도트 아이콘 + 게임 수 뱃지)
    🕵️ 심리전     : 라이어, 마피아, 늑대인간, 인디언 포커
    ⚡ 스피드      : 과일 종!, 폭탄 돌리기, 버저 퀴즈, 이마 퀴즈, 초성 퀴즈
    🎨 그림        : 그림 릴레이, 그림 퀴즈
    🎲 보드·두뇌   : 몽골 대장정 2.0, 오목, 주사위 배팅, 숫자야구
    💬 토크·퀴즈   : 밸런스 게임, 텔레파시, 복불복 룰렛, 상식퀴즈
    📜 스토리      : 텍스트 어드벤처 "고비의 별"
```
### 3-2. 규칙
- 카테고리 펼침 상태·즐겨찾기·최근 게임 = localStorage (`sn_prefs_v1` 단일 JSON 키).
- 게임 카드에 마스코트 스프라이트 + 인원 뱃지 + 예상 소요시간 뱃지 추가 (예: `⏱️10분`).
- 검색은 넣지 않는다 (20개 규모에 과함 — 필터+카테고리로 충분).

- 검수 기준: 정답 유일성, 문항당 40자 이내, 보기 중복 금지.

**수용 기준**
- [ ] 골든벨 전원 탈락 시 부활전 → 직전 생존자 우승 로직
- [ ] 세션 내 문제 중복 0
- [ ] 선택 분포("A 3명, B 1명...")가 공개 화면에 표시

### 4-3. 📜 텍스트 어드벤처 "고비의 별" — 접두어 `ta` (게임 마스터 시스템)

**컨셉**: 도트 정령 GM **텡그리**가 진행하는 파티 협동 TRPG풍 분기 어드벤처. 사전 저작 스토리 그래프 (오프라인이므로 AI 아님 — 전 분기 수작업 저작).

**세계관/시놉시스**: 여행 첫날 밤, 별똥별이 고비 사막에 떨어진다. 일행은 잃어버린 "고비의 별"을 찾아 3일간의 여정을 떠난다. (몽골 실제 소재 활용: 게르, 아이락, 오보, 독수리, 모래폭풍, 샤가이, 마두금, 늑대, 별자리)

**시스템**
- 파티 공유 스탯: `용기 / 지혜 / 인심` 각 2로 시작 (0~5), 자원: `🐑 양 5마리`.
- 프롤로그에서 스탯 배분 선택지 3개 중 다수결 (예: "말을 타고 질주한다(용기+1)" 등).
- **판정**: 선택지에 `check:{stat, dc}` 붙은 경우 → 화면에 픽셀 주사위 d6 굴림 연출 → `스탯 + d6 ≥ dc` 성공. 실패해도 게임오버가 아니라 **다른 경로로 분기** (자원 손실/우회). 게임오버는 최종장 1곳만.
- **선택권 로테이션**: 선택지마다 "이번 선택 담당: {로스터 이름}" 표시 (라운드 로빈). GM 대사로 "파티와 상의해도 좋다" 안내. → 전원 참여 강제 장치.
- GM 연출: 타자기 효과(글자당 18ms, 탭하면 스킵), 텡그리 초상 스프라이트 + 대화창. GM 말투: 신비롭지만 츤데레 ("...제법이군, 인간들.").
- 세이브: 노드 id + 스탯을 localStorage `ta_save_v1`에 자동 저장, 홈에서 "이어하기".

**콘텐츠 분량**
- 3챕터, 노드 총 80~100개, 엔딩 4종:
  ⭐ 진엔딩(별 회수+스탯 조건) / 🐑 목동엔딩(양 10마리 이상) / 🌪️ 방랑엔딩(주요 판정 다수 실패) / 💀 게임오버(최종장 한정, 직전 체크포인트 재시작 제공)
- 플레이타임 목표 25~40분.

**노드 스키마**
```js
// TA_STORY = { nodeId: {...} }
{
  sp: "tengri",              // 화자 스프라이트 키 (tengri|narr|npc_*)
  text: "모래폭풍이 다가온다...",
  set: { sheep:-1 },          // 진입 시 자원/스탯 변동 (선택)
  choices: [
    { label:"게르로 뛴다", goto:"c2_05" },
    { label:"오보 뒤에 숨는다", check:{stat:"지혜", dc:6}, ok:"c2_06", fail:"c2_07" },
    { label:"🐑 양 1마리로 유목민에게 도움 요청", cost:{sheep:1}, goto:"c2_08", hideIf:"sheep<1" }
  ]
}
```
**저작 가이드 (스토리 에이전트용)**
- 노드당 텍스트 90자 이내 (모바일 3~4줄). 유머 3 : 긴장 1 비율.
- 각 챕터에 미니 이벤트 1개씩 기존 게임 크로스오버 (예: "유목민이 샤가이 대결을 청한다 → 즉석 가위바위보로 대체 판정").
- 실존 지명/문화는 존중하는 톤, 초자연 요소는 정령·별 한정.
- 전 노드 도달 가능성 검증 스크립트 필수 (고아 노드/무한루프/미정의 goto 0건).

**수용 기준**
- [ ] 세이브/이어하기/처음부터 동작
- [ ] 판정 주사위 연출 + 실패 분기 존재 (즉사 없음)
- [ ] 선택권 로테이션에 로스터 이름 표시
- [ ] 그래프 검증 스크립트 통과 (도달 불가 노드 0)

## 5. 📡 연결 가이드 페이지 (앱 내 정적 페이지, 접두어 `guide`)

홈 상단 [📡 연결 가이드] → 도트 일러스트 스텝 카드로 구성:

1. **여행 전 준비**: "와이파이 있을 때 전원 이 주소 접속 → 설치" (QR코드를 앱이 자체 생성해 표시 — 자기 URL의 QR, 라이브러리 없이 canvas QR 구현 or 사전 생성 스프라이트).
2. **iOS 설치법** 3스텝 / **Android 설치법** 2스텝 일러스트.
3. **현지에서**: "인터넷 없어도 앱 아이콘으로 실행하면 끝. 폰 1대로 돌려가며 플레이."
4. **핫스팟은 언제 쓰나요?**: 정직하게 —
   - "지금 버전은 서로 연결이 필요 없어요. 한 폰으로 충분!"
   - "일행 중 설치 못 한 사람이 있으면: 설치한 사람이 **핫스팟**을 켜고 → (호스트가 데이터 여유 있을 때) 접속시켜 설치. 현지 유심 1개만 있어도 전원 설치 가능."
5. **자주 묻는 것**: 배터리 절약 팁, 화면 자동잠금 끄기 안내, 업데이트 방법.

> **v3 로드맵 (기획서에만 기록, 이번 구현 제외)**: 진짜 폰 N대 실시간 멀티 = 오프라인 WebRTC + QR 수동 시그널링 실험안. 호스트가 offer QR 표시 → 게스트 스캔 → answer QR 역스캔. 기술적으로 가능하나 N명 페어링 UX 부담이 커서 보류. 만약 착수 시 대상 게임: 버저 퀴즈(각자 폰이 버저), 텔레파시(동시 입력).
