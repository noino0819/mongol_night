# 방탈출(er) 출시 전 체크리스트

새 시나리오·퍼즐·대화를 만들거나 고칠 때마다 **출시 직전 이 리스트를 처음부터 끝까지 훑는다.** 원칙의 *원전*은 아래 문서들이고, 이 파일은 그걸 "빠뜨리기 쉬운 실수" 형태로 압축한 **재확인 게이트**다. 항목이 하나라도 걸리면 재설계.

**함께 볼 원전**: `docs/방탈출_트릭사전.md`(0장 불변 원칙 + 트릭 카탈로그 + 몽골 팩트 부록) · `docs/방탈출게임_리서치.md`(설계 원칙 원전) · 메모리 `er-design-directives`(유저 확정 지시).

> 판정 한 줄: 플레이어 입에서 **"그걸 어떻게 알아?"** 가 나오면 너무 어렵고, **"아 그냥 다 눌렀더니 풀렸네"** 가 나오면 떠먹인 거다. 목표는 **"아 그거였구나!"**.

---

## A. 반전·정체 (범인/흑막이 있는 시나리오)
- [ ] **반전·정체를 미리 알려주지 않는다.** 흑막이 누군지, 그를 식별하는 *규칙*(예: "상태창 안 뜨는 자가 흑막")을 소개문·NPC 지문·시스템 메시지에서 **먼저 말하지 않는다.**
- [ ] **정체 규칙은 딱 한 곳에서만** 준다. 여러 오브젝트에서 같은 규칙을 반복 서술하면 추리가 사라진다. 그 한 곳도 **게이트 뒤(금고·서고 안쪽)** 에 두어 "얻어낸 단서"로 만든다.
- [ ] **관찰 → 규칙 → 지목**이 분리돼 있다. 플레이어가 *관찰*(누가 이상한지)은 스스로 발견하고, *규칙*(이상한 자=범인)은 단서로 얻고, *지목*(그래서 누구)은 직접 추론한다. 셋 중 하나라도 지문이 통째로 대신 해 주면 실패.
- [ ] 반전 힌트를 초반 이스터에그(taps 등)에 넣지 않는다. "흑막은 이미 네 앞에 있던 자" 류의 조기 스포 금지.

> 반례(수정 전 회귀자): 연회장 소개문이 "한 명은 상태창이 안 뜬다"를 대놓고 말하고, 같은 규칙을 일지·시스템 reveal·NPC 지문 네 곳에서 반복 → 추리 소멸. → 관찰은 대화로, 규칙은 금고 안 일지 한 곳으로 분리해 수정.

## B. 대화·NPC
- [ ] **대화는 정적 나열이 아니라 인터랙션이다.** 여러 NPC 대사를 한 화면에 통째로 뿌리지 않는다. **초상을 눌러** 한 명씩 말을 붙이게 한다.
- [ ] **핵심 단서를 대화 여는 즉시 노출하지 않는다.** 첫 대사는 잡담·분위기. 결정적 정보는 **뒤쪽 대사**에 숨겨, 같은 NPC와 여러 번 말해야 나오게 한다(`lines:[...]`).
- [ ] 결정적 단서를 흘리는 NPC 말고, **다른 NPC가 "그쪽을 봐라"** 정도의 부드러운 유도만 준다(지목은 금지).
- [ ] 대사가 있으면 "더 눌러서 말 붙이기" 같은 **이어짐 표시**가 마지막 대사에서 사라진다(끝났음을 알 수 있게).

## B-2. 말하지 말고 보여줘라 (show, don't tell)
- [ ] **관찰형 단서를 지문으로 서술하지 않는다.** "…상태창이 뜨지 않는다", "…뭔가 이상하다" 처럼 위화감을 글로 대신 말해 주지 말 것.
- [ ] **실제 시각 이펙트·인터랙션으로 만든다.** 그 인물과 대화하면 상태창 UI가 진짜로 흔들리고 숫자가 안 뜨게(`er-glitch`), 정상 대상과 나란히 두어 **플레이어가 눈으로 위화감을 캐치**하게.
- [ ] **반응할 여지를 남긴다.** 스탯·변경·이상은 이펙트+소리로, 유저가 직접 발견하고 "어?" 하게. (메모리 `stat-changes-as-effects`와 같은 원칙 — 서술로 떠먹인 위화감보다 눈으로 잡은 위화감이 재밌다.)

## C. 퍼즐 난이도 (떠먹임/노가다 방지)
- [ ] **"다 누르면 풀리는" 퍼즐이 없다.** 수집(뒤집기·비추기·헤치기)과 해답(규칙 적용)이 분리돼 있다. 셀을 다 까기만 하면 답이 되는 구조 금지.
- [ ] **단서→자물쇠 연결을 텍스트가 대신 이어주지 않는다.** 연결은 **태깅**(색·문양·재질·이름 일치)으로만 암시. "이 숫자를 저 자물쇠가 원한다" 금지.
- [ ] **find/torch 프롬프트가 답을 지목하지 않는다.** "가장 오래된 해에 이름 남긴 사람이 범인" 처럼 규칙+답을 프롬프트가 통째로 말하면, 관찰 인터랙션이 장식이 된다. 프롬프트는 *무엇을 하는지*만, *무엇이 답인지*는 플레이어가.
- [ ] **셀·표식 라벨에 답을 적어두지 않는다.** "4:26(멎음)" 처럼 정답 셀에 "(멎음)"을 붙이면 뒤집는 순간 끝. 구분은 스프라이트/맥락으로.
- [ ] **규칙 지식(치환표·순서규칙)은 방 안에서 준다.** 외부 상식(모스부호 암기 등) 요구 금지. 단, 규칙을 주는 것 자체는 떠먹임이 아니다 — 일은 규칙의 *발견과 적용*에 있게.
- [ ] **개수 세기**는 한 자리 수·명확한 대상·명확한 범위만.
- [ ] **첫 퍼즐은 3~5분 내** 풀리고, 복잡한 연쇄는 후반에.

## D. 힌트 (3단, 부드럽게)
- [ ] 3단 힌트 = **1단 부드러운 넌지**(어디를 생각할지) → 2단 지목 → 3단 답.
- [ ] **1단이 방법·답을 바로 말하지 않는다.** (2·3단은 진짜 막힌 사람 구제용으로 유지)

## E. 숨은 인터랙션 (taps 등)
- [ ] **신호 없이·아껴서.** "여기를 눌러 봐" 안내 금지. 원래 스프라이트를 눌렀을 때 흔들림만 주고, 신호 없이 N번 눌러야 발견되게.
- [ ] 시나리오당 가끔만(이스터에그). 남발 금지.

## F. 기술 검증 (매번)
- [ ] `node --check games/er.js` — 문법 OK
- [ ] `node tools/back-test.mjs` — 뒤로가기/닫기 가드 OK
- [ ] 신규 스프라이트 있으면 `node tools/sprite-lint.mjs` (제작 법칙: `docs/도트스프라이트_가이드.md`)
- [ ] 판정·정산 로직 바꿨으면 node로 추출해 단위 케이스 3개 이상 통과.
- [ ] 사용자 입력 → innerHTML 경로에 `escHtml()` 적용.
- [ ] 새 상태 필드는 `erBeginAct` 리셋 + `erFixSave` 기본값에 추가(이어하기 세이브 호환).
- [ ] 리셋 함수가 자기 타이머·스트림을 전부 정리(화면 이탈 시 계속 돌지 않게).
- [ ] 시각 효과(글리치 등)는 **메뉴얼 모드**로 눈으로 확인 — 진입: 홈 로고 7연타 또는 설정 화면 버전 푸터 10탭. "게임 직행"으로 일행 없이 바로 입장, "효과 프리뷰"로 상태창 정상/글리치 대조.

## G. 톤·표기
- [ ] 카피는 가벼운 반말. **ㅋㅋ·ㅎㅎ 등 자모 웃음 금지.**
- [ ] 버튼은 동사 라벨.
- [ ] 장식 이모지 지양, 시각 요소는 도트 스프라이트로.
- [ ] 스토리 필수 — 퍼즐 소재가 그 세계관에서 자연스럽게 나온 것. 시대착오 금지(트릭사전 부록 A 팩트만).
