## 통합 가이드 (클로드코드용)

1. 이 파일의 13개 카테고리를 기획서의 카테고리 테이블에 **추가**한다 (기존 8종 + 13종 = 21종, 총 1000문항).
2. 카테고리가 21개이므로 선택 UI는 **그룹 접기** 권장: [클래식 8종] / [확장 13종] 두 섹션으로 나누고, "전체 선택 / 전체 해제" 버튼 추가.
3. ID 규칙·중복 방지 로직(localStorage `balance_seen_v1`)·소진 처리 등은 기획서 2장을 그대로 적용. **기존에 본 질문 기록은 그대로 유지되어야 함** (스키마 변경 아님 → 저장 키 버전 올리지 말 것).
4. 파싱: 각 코드블록의 `id | A | B` 라인을 정규식 `^([A-Z0-9]{2}\d{2}) \| (.+) \| (.+)$`로 추출하여 JS 배열 생성.
5. 검수 기준: 파싱된 문항 수 = 정확히 800개. 기존 200개와 id 충돌 없음 (접두어가 전부 다름).
