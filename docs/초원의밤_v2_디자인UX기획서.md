- 이름: **초원의 밤** / 영문 slug: `steppe-night`

## 2. 앱 셸 화면 플로우

```
[브라우저 첫 방문]
스플래시(≤1.2s) → 온보딩(첫 방문만, 4장) → 설치 유도 시트 → 홈
[설치된 PWA 실행]
OS 스플래시(manifest) → 홈 (온보딩 스킵)
```

### 2-1. 스플래시
- 웹 최초 로드용 인앱 스플래시: 배경 `--night`, 중앙 로고 세로 조합, 아래 픽셀 프로그레스 바(폰트·에셋 파싱 동안). 최대 1.2초, 준비 완료 시 페이드 아웃 200ms.
- PWA 설치 후에는 manifest(background_color+icon) 기반 OS 스플래시가 대체 → 인앱 스플래시는 `standalone`에서 표시 시간 0.4초로 단축.

### 2-2. 온보딩 (첫 방문 1회, `localStorage sn_prefs_v1.onboarded`)
- 형식: 풀스크린 4장 가로 스와이프 카드 + 하단 픽셀 도트 인디케이터 + [건너뛰기].
- 각 장 = 도트 일러스트(스프라이트 조합 씬) + 헤드라인 1줄 + 서브 1줄:
  1. 🌌 "와이파이? 필요 없어" — 게르 앞 모닥불에 모인 동물들 / "비행기 모드에서도 전부 돌아가는 파티게임 20종"
  2. 📱 "폰 하나면 충분" — 폰을 돌려받는 양과 여우 / "돌려가며 하는 패스앤플레이. 비밀은 꾹 눌러 확인!"
  3. 🎲 "가볍게도, 진하게도" — 주사위·카드·붓 아이콘 / "5분 폭탄 돌리기부터 40분 어드벤처까지"
  4. 📲 "홈 화면에 설치하면 완성" — 다운로드 화살표 받는 보르 / CTA 버튼: [앱으로 설치하기] (iOS면 설치 안내 시트 오픈) + [나중에]
- 마지막 장 완료 → 로스터가 비어 있으면 홈 진입 시 "일행 등록" 코치마크 1회 표시(아래 6-3).

### 2-3. 설치 유도 시트 (바텀시트)
- Android: `beforeinstallprompt` 보유 시 [설치] 네이티브 프롬프트 트리거.
- iOS: 3스텝 도트 일러스트 (공유 버튼 → "홈 화면에 추가" → 완료) + "사파리에서만 가능해요" 뱃지.
- 닫으면 홈 상단에 작은 [📲 설치] 칩으로 축소 잔존. standalone 실행 중이면 전부 숨김.

## 4. 로딩·스켈레톤·상태 디자인

### 4-1. 폰트 로딩 (FOUT 방지)
- base64 인라인이므로 네트워크 대기는 없지만 파싱 동안 시스템 폰트 플래시 가능 → `font-display: block` + 스플래시가 파싱 시간을 흡수. 스플래시 해제 조건에 `document.fonts.ready` 포함.

### 4-2. 스켈레톤 UI (도트 스타일로 재해석)
- 일반 shimmer 대신 **픽셀 스켈레톤**: 4px 그리드 회색 블록이 2프레임으로 깜빡(체커보드 ↔ 반전, 500ms). 클래스 `.px-skel`.
- 적용 지점:
  - 홈 최초 진입: 카테고리 카드 6개 자리 (SW/폰트 준비 동안, 보통 0.5초 미만이라도 레이아웃 시프트 방지 목적)
  - 그림 게임 변천사 공개: dataURL 이미지 디코드 동안 캔버스 자리
  - TRPG 챕터 전환: 대화창 자리
- 규칙: 300ms 미만 완료 시 스켈레톤 생략(깜빡임 방지, 지연 표시 패턴).

### 4-3. 상태 4종 표준 컴포넌트
| 상태 | 디자인 |
|---|---|
| 빈 상태 | 마스코트 스프라이트(반투명) + 1줄 안내 + CTA. 예: 로스터 0명 홈 → 보르가 두리번 + "일행을 등록하면 시작!" [+ 등록] |
| 오프라인 준비 | 홈 상단 배지: ⏳(다운로드 중, 프로그레스) → ✅(완료) → ⚠️(SW 실패: "새로고침 해주세요") |
| 에러 | RPG 대화창 톤: "텡그리: ...이건 예상 밖이군. [다시 시도]" (스택트레이스 노출 금지) |
| 업데이트 | 하단 토스트 "🆕 새 버전 도착! 탭해서 적용" (지속 표시, 스와이프로 닫기 가능) |

## 5. 마이크로 인터랙션 (전역 규격)

| 요소 | 스펙 |
|---|---|
| 버튼 눌림 | translateY(4px) + 하단 픽셀 그림자 제거, 80ms, easing 없음(스텝 전환이 도트 감성) |
| 화면 전환 | 200ms 스텝 페이드 (opacity 0→1을 4단계 계단으로: `steps(4)`) |
| 햅틱 | `navigator.vibrate` 지원 시: 버튼 10ms / 정답·승리 [30,40,30] / 폭탄·패배 [200] . 설정에서 off 가능 |
| 토스트 | 하단, 픽셀 프레임, 2.4초 자동 소멸, 동시 1개(큐잉) |
| 승리 연출 | 마스코트 2프레임 점프 + 픽셀 콘페티(8×8 사각 12개 낙하) + (사운드 on 시) 8비트 팡파레 |
| 카운트다운 | 3-2-1 숫자 스케일 팝(각 1초, steps(2)) — 전 게임 공통 컴포넌트로 통일 |
| 길게 누르기 | 즐겨찾기 등록 시 카드에 픽셀 별 채워지는 4단계 애니 + 햅틱 30ms |

## 6. 화면별 UX 디테일

### 6-1. 홈
- 레이아웃 순서: 상태 배지줄(오프라인/설치/설정/가이드) → 로고 가로 조합 → 인원 필터 칩 → 최근 한 판 → 즐겨찾기 → 카테고리 폴더.
- 카테고리 폴더 오픈: 아코디언 160ms, 열릴 때 폴더 마스코트가 고개 드는 2프레임.
- 게임 카드: 좌측 마스코트 48×48(스프라이트 3배), 우측 이름+한줄 설명+뱃지(인원·시간). 길게 누르면 ⭐.

### 6-2. 게임 공통 셸
- 상단바 통일: [← 카테고리명] + 마스코트 24px + 게임명 + 우측 [ℹ️ 룰]
- **[ℹ️ 룰] 버튼 신설**: 어느 게임에서든 룰 요약 바텀시트 (기존 hint 문구 재사용) — 중간 난입자 온보딩용.
- 뒤로가기 시 진행 중이면 확인 모달 ("판이 날아가요! 나갈까?") — 실수 이탈 방지. 브라우저 뒤로가기(popstate)도 동일 처리.

### 6-3. 코치마크 (1회성 툴팁, `sn_prefs_v1.coach`)
1. 홈 첫 진입: "일행 등록" 입력창 하이라이트
2. 첫 비밀 화면: "꾹 누르는 동안만 보여요" 
3. 첫 게임 카드: "길게 누르면 즐겨찾기 ⭐"
- 규칙: 한 화면에 코치마크 1개만, [알겠어] 탭으로 소멸, 전체 3개 초과 금지.

### 6-4. 설정 페이지 (⚙️)
- 항목: 사운드 on/off(기본 off) / 진동 on/off(기본 on) / 텍스트 속도(TRPG: 보통·빠름·즉시) / 일행 관리(로스터 편집 진입) / 데이터 초기화(확인 2단계) / 버전·오픈소스 폰트 라이선스 표기(Galmuri OFL 고지 — 라이선스상 필수).

## 7. 접근성·가독성 가드레일

- 픽셀 폰트 최소 크기: 본문 14px 이상, 버튼 16px 이상. 11px 미만 사용 금지 (대장정 보드 타일만 예외 8px — 대신 탭하면 타일 상세 팝업 제공하는 것으로 보완).
- 대비: 본문 `--milk` on `--night` 유지. `--dim`은 보조 정보에만.
- 터치 타깃 최소 44×44px (보드 타일 제외 전 인터랙션).
- 애니메이션: `prefers-reduced-motion` 시 콘페티·점프·계단 페이드 비활성(즉시 전환).
- 색만으로 상태 구분 금지: 팀 색상엔 항상 이름/아이콘 병기 (기존 준수 유지).

## 8. 카피라이팅 톤 가이드

- 기본: 친구 말투 반말 + 가끔 ㅋㅋ. 이모지는 문장당 최대 1개.
- 텡그리(GM/에러 화자)만 예외: "~군, ~하지" 신비 츤데레체.
- 버튼은 동사로 끝내기 ("시작!", "다음 사람에게 →"). "확인/취소" 같은 무미건조 라벨 지양 → 맥락 라벨 ("판 접기", "계속 하기").
- 금지: 존댓말·반말 혼용, 과한 느낌표 연타(!!! 금지, ! 하나까지).
