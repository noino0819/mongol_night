# 밸런스 게임 확장판 — 개발 기획서

> 이 문서 하나로 개발이 완결되도록 작성됨. 질문 뱅크 200개 전체 포함.
> 대상: "초원의밤_몽골게임팩.html" 내 기존 밸런스 게임을 이 스펙으로 교체하거나, 단독 HTML 파일로 신규 개발.

---

## 1. 개요

- **목적**: 몽골 여행(6~8인, 오프라인)에서 즐길 밸런스 게임. 질문 200개, 카테고리 8종, **이미 나온 질문은 다시 안 나오는** 중복 방지 기능이 핵심.
- **환경**: 인터넷 없음. 단일 HTML 파일(외부 CDN/폰트/이미지 금지). 모바일 브라우저(Chrome/Safari) 기준, 세로 화면 최적화 (~390px).
- **조작**: 폰 하나를 다같이 보며 진행. 터치만 사용.

## 2. 핵심 기능 명세

### 2.1 질문 데이터 스키마

```js
// 질문 1건
{ id: "FD01", cat: "food", a: "짜장면", b: "짬뽕" }
```

- `id`: 카테고리 접두어 + 2자리 번호. **한 번 배포된 id는 절대 변경/재사용 금지** (중복 방지 기록의 키이기 때문).
- 질문 추가 시 새 id를 뒤에 붙이기만 한다. 삭제 시에도 id는 결번 처리.

### 2.2 카테고리 (8종)

| key | 이름 | 개수 | 아이콘 |
|---|---|---|---|
| food | 음식 클래식 | 25 | 🍜 |
| fun | 웃긴/엽기 | 30 | 🤪 |
| love | 연애 | 25 | 💘 |
| friend | 우정/단체생활 | 20 | 🤝 |
| work | 직장/돈 | 25 | 💼 |
| power | 초능력/상상 | 25 | 🦸 |
| extreme | 극한선택/생존 | 20 | 🔥 |
| mongol | 여행/몽골 스페셜 | 30 | 🐪 |

- 카테고리 다중 선택 가능(토글 칩). 기본값: 전체 선택.
- "전체 랜덤" = 선택된 카테고리들의 미출제 질문을 합쳐서 셔플.

### 2.3 중복 방지 (핵심 기능)

**요구사항**: 한 번 나온 질문은 기기 기준으로 다시 나오지 않아야 함. 앱을 껐다 켜도 유지.

**구현**:

```js
const STORE_KEY = "balance_seen_v1";   // 스키마 바뀌면 v2로 올림

function loadSeen() {
  try { return new Set(JSON.parse(localStorage.getItem(STORE_KEY) || "[]")); }
  catch (e) { return new Set(); }      // localStorage 불가 환경 → 세션 한정 메모리로 동작
}
function saveSeen(seen) {
  try { localStorage.setItem(STORE_KEY, JSON.stringify([...seen])); }
  catch (e) { /* 무시: 메모리 모드로 계속 */ }
}
```

- **기록 시점**: 질문이 화면에 "표시된 순간" seen에 추가하고 즉시 저장. (다음 버튼 누를 때가 아님 — 중간에 꺼져도 그 질문은 소진된 것으로 처리)
- **덱 구성**: 게임 시작/카테고리 변경 시 `선택 카테고리 질문 − seen` 을 셔플하여 덱 생성. `pop()`으로 출제.
- **전부 소진 시**: "선택한 카테고리의 질문을 모두 봤어요! 🎉" 화면 표시 + 선택지 2개 제공:
  1. `[본 질문 다시 섞기]` → 해당 카테고리의 seen만 제거 후 재시작
  2. `[다른 카테고리 고르기]` → 카테고리 선택 화면으로
- **남은 개수 표시**: 카테고리 칩에 `음식 클래식 (12/25)` 형태로 미출제/전체 표시. 진행 중 상단에도 "남은 질문 37개" 뱃지.
- **전체 초기화**: 설정 구석에 작게 `[기록 전체 초기화]` (confirm 필수).
- **주의**: localStorage는 파일을 여는 방식(file:// 경로)이 바뀌면 별도 저장소가 됨 → 여행 전 같은 방식으로 열어서 테스트하라는 안내 문구를 최초 실행 시 1회 노출.

### 2.4 게임 진행 UI

```
[카테고리 선택 화면]
  ┌─────────────────────────┐
  │ 카테고리 칩 (다중 토글)   │  ← 각 칩에 (남은/전체) 표시
  │ [게임 시작] 버튼          │
  └─────────────────────────┘

[플레이 화면]
  ┌─────────────────────────┐
  │ 🍜 음식 클래식  남은 37개 │  ← 현재 질문의 카테고리 + 남은 수
  │ ┌─────────────────────┐ │
  │ │   A 선택지 (큰 버튼)  │ │  ← 탭하면 하이라이트 (재미용, 다수결은 손으로)
  │ └─────────────────────┘ │
  │         V S             │
  │ ┌─────────────────────┐ │
  │ │   B 선택지 (큰 버튼)  │ │
  │ └─────────────────────┘ │
  │ [이유 타이머 30초] [다음 →] │
  └─────────────────────────┘
```

- **이유 타이머(선택 기능)**: 소수파가 왜 그걸 골랐는지 30초 안에 변론하는 타이머 버튼. 사이트들 공통 꿀팁("왜?를 물어보면 대화가 터진다") 반영.
- 다음 버튼을 누르면 새 질문. 뒤로가기(이전 질문 다시 보기)는 **미지원** — 중복 방지 로직 단순화를 위해.

### 2.5 V2 (여유 있으면)

- 선택 기록 통계: 각 질문에서 A/B 탭 횟수를 기록 → "이번 여행 최다 만장일치 질문 / 최다 접전 질문" 결산 화면.
- 커스텀 질문 추가: 사용자가 직접 입력(id는 `CUSTOM_타임스탬프`), localStorage 별도 키에 저장.

## 3. 수용 기준 (Acceptance Criteria)

1. 비행기 모드에서 모든 기능 정상 동작 (외부 요청 0건 — DevTools Network 탭으로 검증).
2. 같은 질문이 두 번 표시되지 않음 (새로고침/재실행 포함, localStorage 동작 환경 기준).
3. 200문항 전부 소진 시 소진 화면이 정상 표시되고, 재섞기 후 다시 200개가 나옴.
4. localStorage가 막힌 환경(일부 iOS file:// 등)에서도 크래시 없이 세션 내 중복 방지로 동작.
5. 390px 폭에서 선택지 텍스트가 잘리지 않음 (최장 문항 기준으로 확인).

## 5. 개발 참고: 기존 게임팩에 통합하는 경우

- 기존 파일의 `BALANCE` 배열(2차원 배열)과 `balNext()` 로직을 이 스펙의 `{id, cat, a, b}` 구조로 교체.
- 기존 게임팩은 **claude.ai 아티팩트가 아니라 로컬 HTML 파일로 배포**되므로 localStorage 사용 가능. 단, 2.3의 try-catch 폴백은 반드시 유지.
- 카테고리 선택 UI는 기존 `.seg` 칩 컴포넌트 재사용.
- 색상/폰트 토큰은 기존 `:root` CSS 변수(--night, --fire, --milk 등)를 그대로 따를 것.

## 6. 테스트 시나리오

1. 카테고리 1개(food)만 선택 → 25문항 연속 출제 → 26번째에 소진 화면 → [다시 섞기] → 다시 25개.
2. 진행 중 브라우저 강제 종료 → 재실행 → 이미 본 질문 안 나옴 + 남은 개수 정확.
3. 전체 카테고리 선택 → 남은 질문 수 = 200 − seen 개수와 일치.
4. [기록 전체 초기화] → confirm 취소 시 유지, 확인 시 모든 카운트 리셋.
5. 시크릿 모드(localStorage 제한 가능) → 크래시 없이 동작, 세션 내 중복 없음.
