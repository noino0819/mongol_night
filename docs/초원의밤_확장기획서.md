# 초원의 밤 — 몽골 게임팩 확장 기획서 (Claude Code 서브에이전트용)

> 대상 파일: `초원의밤_몽골게임팩.html` (단일 파일, 현재 15개 게임 구현 완료)
> 이 문서는 (A) 기존 아키텍처 문서화, (B) 신규 게임 3종 상세 스펙, (C) 서브에이전트 작업 규칙, (D) QA 기준으로 구성된다.
> 별도 문서: `밸런스게임_기획서.md`(200문항+중복방지 스펙), `밸런스게임_추가800.md`(800문항 데이터) — 밸런스 게임 관련 작업은 그 문서를 우선한다.

## 0. 제품 제약 (절대 위반 금지)

1. **단일 HTML 파일** — 외부 CDN/폰트/이미지 요청 0건. 인터넷 0%에서 동작.
2. **패스앤플레이** — 폰 1대를 돌려가며 플레이. 네트워크 통신 없음.
3. 모바일 세로 최적화: 컨테이너 max-width 430px, 뷰포트 ~394px 기준.
4. localStorage 사용 금지 (밸런스 게임 중복 방지 기능만 예외 — 해당 기획서 참조). 나머지는 전부 인메모리.
5. iOS Safari / Android Chrome 양쪽 호환. `pointerdown/up/cancel/leave` 사용 (touch 이벤트 직접 사용 금지), `contextmenu` 방지 필수(꾹 누르기 UI).
6. 한국어 UI, 반말+ㅋㅋ 톤 유지.

- 화면 = `<div class="screen" id="scr-{게임키}">`. `data-go="{게임키}"` 버튼으로 전환.

## B. 신규 게임 상세 스펙 (미구현 3종)

### B-1. 🎰 주사위 배팅 — 접두어 `lv` (라스베가스 스타일)

**컨셉**: 상금 더미에 주사위로 눈치 배팅. 최다 배치자가 독식하되 **동률은 전원 무효**가 킬포인트.

**셋업**
- 참여자 2~5명 (로스터 칩). 6명 이상 선택 시 alert "5명 초과는 2인 1팀 추천!".
- 라운드 수 seg: 2 / 3(기본) / 4.

**라운드 구조**
1. 상금 슬롯 4개 생성. 슬롯 번호 = 3,4,5,6 (주사위 눈 3~6에 대응. 1~2는 미사용 → 굴린 1·2 눈은 "꽝, 버려짐" 처리해 템포 업).
   - 슬롯별 상금: 3~8양 랜덤. 상금 합이 14 미만이면 리롤.
2. 각 플레이어 주사위 7개 보유. 현재 플레이어 턴:
   - [🎲 굴리기] → 남은 주사위 전부 굴림 → 눈별 개수 표시 (예: 3×2개, 4×1개, 5×3개, 꽝(1·2)×1개).
   - 꽝 주사위는 자동 소각. 3~6 중 **한 눈을 선택** → 그 눈의 주사위 전부를 해당 슬롯에 배치.
   - 남은 주사위 있으면 다음 사람으로 (라운드 로빈). 전원 소진 시 정산.
3. **정산**: 슬롯별로 배치 수 1위가 상금 획득. **1위 동률이면 그들 전원 제외하고 차순위가 획득** (차순위도 동률이면 반복, 전원 동률이면 상금 소멸). 결과를 슬롯별 lifeModal 또는 목록으로 표시.
4. 다음 라운드 (주사위·슬롯 리셋). 최종: 총 양 랭킹.

**상태 스키마**
```js
let lv = { sel:[], rounds:3, round:1, slots:[], // [{prize, placed:{playerIdx:count}}] index 0→눈3
  p:[], // [{name, dice:7, sheep:0}]
  turn:0, lastRoll:null };
```
**UI**: 슬롯 4장 가로 카드(상금 크게 + 배치 현황 색점), 아래 굴림 결과 + 눈 선택 버튼 4개(개수 뱃지), `.mb-strip`으로 잔여 주사위/양 표시.

**수용 기준**
- [ ] 꽝(1·2) 자동 소각이 명시적으로 표시된다
- [ ] 동률 무효 규칙이 정확히 동작한다 (전원 동률 → 상금 소멸 포함)
- [ ] 주사위 전원 소진 시 자동 정산된다
- [ ] 라운드 간 양은 누적, 주사위/슬롯은 리셋

### B-2. 🔔 버저 퀴즈쇼 — 접두어 `bz`

**컨셉**: 폰을 테이블 중앙에 놓고, 화면 모서리 버저를 먼저 탭한 사람이 답할 권리 획득.

**셋업**
- 버저 인원 2~4명 (로스터에서 선택, 화면 모서리에 배치. 5명 이상이면 2인 1팀 안내).
- 문제 유형 multi seg: [OX 상식] [초성 퀴즈] [빠른 계산] (복수 선택, 최소 1).
- 목표 점수 seg: 5 / 7(기본) / 10점 선취.

**문제 뱅크 (파일 내 상수로 신규 작성)**
- `BZ_OX`: 40문항 `{q, a:true|false}` (예: "몽골의 수도는 울란바토르다 → O", "낙타 혹에는 물이 들어있다 → X"). 상식은 논란 없는 것만.
- 초성: 기존 `CHO_BANK` 재사용 (정답 문자열 보유).
- `BZ_CALC`: 런타임 생성 — `a op b` (a,b: 2~30, op: +,−,×), × 는 한 자리수끼리.

**플레이 루프**
1. 중앙에 카운트다운 3→1 후 문제 표시. 버저 존 4개(2인이면 상/하 분할)는 회전 배치(각자 자기 방향에서 보이게 `transform: rotate`).
2. 첫 버저 탭 감지 → 다른 버저 잠금 + 해당 존 하이라이트 + 5초 답변 타이머.
3. 판정:
   - OX: 화면 중앙에 [⭕][❌] 버튼 → 누른 값과 정답 자동 비교.
   - 계산: 정답 숫자를 중앙 [정답 공개] 후 심판 판정 [맞음][틀림].
   - 초성: [정답 공개(꾹)] 후 심판 판정 [맞음][틀림].
   - 맞음 +1, 틀림 −1(최저 0) 후 **같은 문제로 나머지 인원 재버저** (오답자는 그 문제 잠금).
4. 목표 점수 도달 시 승리 화면. 점수는 각 존 옆에 상시 표시.

**상태 스키마**
```js
let bz = { sel:[], types:["ox"], goal:7, scores:[], cur:null, locked:[], phase:"idle", tid:null };
```
**수용 기준**
- [ ] 버저 동시 탭 시 pointerdown 타임스탬프 빠른 쪽 1개만 활성
- [ ] 오답자는 해당 문제 재버저 불가
- [ ] 존 텍스트가 각 플레이어 방향으로 회전되어 있다
- [ ] 문제 유형 복수 선택 시 랜덤 믹스 출제

### B-3. ⚾ 숫자야구 — 접두어 `nb`

**컨셉**: 3자리 비밀 숫자 상호 추리. 이동 중 차 안에서 2인 대전용.

**셋업**
- 모드 seg: [1:1 대전(기본)] [팀전 2:2].
- 자릿수 seg: 3자리(기본) / 4자리. 규칙: 0~9 중복 없음, 맨 앞 0 허용.

**플로우**
1. 비밀 설정: 패스 단계 → A가 몰래 숫자 입력(유효성: 자릿수·중복 검사) → B도 입력.
2. 교대 추측: 현재 추측자 화면에 숫자패드 입력 → 제출 시 자동 판정 `nS mB` (자리·숫자 일치=S, 숫자만=B, 0이면 OUT). 양쪽 추측 로그를 좌우 2열로 상시 표시 (자기 로그 + 상대 로그. 상대의 비밀번호는 당연히 비표시).
3. 승리: 올 스트라이크. **후공에게 마지막 1회 동수 기회** 부여(선공 승리 시) → 후공도 맞히면 무승부 처리.
4. 종료 화면: 비밀 숫자 공개 + 시도 횟수.

**상태 스키마**
```js
let nb = { digits:3, secret:["",""], turn:0, logs:[[],[]], phase:"set", firstSolver:-1 };
```
**수용 기준**
- [ ] 중복 숫자/자릿수 미달 입력 거부
- [ ] S/B 판정 정확 (동일 숫자 재사용 케이스 포함 단위 검증)
- [ ] 선공 승리 시 후공 마지막 기회 → 무승부 지원
- [ ] 로그가 추측자 본인 화면에서만 노출되는 것이 아니라 양쪽 모두 공개 (비밀 숫자만 숨김)

## C. 서브에이전트 작업 규칙

### C-1. 작업 분할
- 에이전트 1개 = 게임 1개. 병렬 실행 시 **파일 충돌 방지를 위해 삽입 지점을 분리**한다:
  - 홈 카드: 기존 마지막 `.game-card` 뒤에 자기 카드 1개만 append
  - 화면 HTML: `<!-- ============ 룰렛 ============ -->` 주석 **앞**에 자기 화면 1블록만 삽입
  - CSS: `</style>` 직전에 자기 블록(주석 헤더 필수 `/* ---------- {게임명} ---------- */`)
  - JS: `</script>` 직전에 자기 블록 append. 기존 함수/전역은 **읽기만** 하고 수정 금지 (need 맵, resetGame 스위치 2곳만 예외적 수정 허용)
- 순차 실행이 가능하면 순차를 우선한다 (같은 파일 병렬 편집은 머지 충돌 리스크).

### C-2. 공통 구현 체크리스트 (게임당)
- [ ] need 맵 + resetGame 훅 + 홈 카드 + scr-{키} 화면 등록
- [ ] Reset 함수에서 모든 setInterval/setTimeout clear
- [ ] 사용자 입력 → innerHTML 경로는 `escHtml` 필수
- [ ] 비밀 정보는 패스 단계 뒤에만 노출
- [ ] 색상은 CSS 변수만, 새 하드코딩 색 금지
- [ ] 이모지·이름 등 상수는 파일 내 선언 (외부 요청 금지)

### C-3. 검증 스크립트 (구현 후 필수 실행)
```bash
node -e "
const fs=require('fs');
const html=fs.readFileSync('초원의밤_몽골게임팩.html','utf8');
const m=html.match(/<script>([\s\S]*)<\/script>/);
new Function(m[1]);                       // 문법
console.log('JS OK', m[1].length);
console.log('cards', (html.match(/class=\"game-card\"/g)||[]).length);
"
```
추가로 게임별 순수 로직(판정/정산 함수)은 위 스크립트처럼 node로 추출해 단위 케이스 3개 이상 통과시킬 것 (예: 주사위 배팅 동률 무효, 숫자야구 S/B, 버저 동시입력).

### C-4. 밸런스 게임 통합 작업 (별도 에이전트)

## D. QA 시나리오 (사람 손 테스트, 출국 전 1회)

1. **파일 오픈**: 안드로이드 Chrome에서 file:// 열기 → 홈 렌더 확인. iOS는 파일앱 미리보기에서 JS 미실행 가능 → Safari로 열기 확인.
2. 로스터 8명 등록 → 15+3개 게임 전부 진입/복귀 (뒤로가기 시 타이머 잔존 버그 체크: 다른 게임 화면에서 이전 게임 팝업이 뜨면 실패).
3. 늑대인간 5인 1판: 강도+말썽꾸러기 둘 다 발동시키고 결과 화면 역할 변화 표기 확인.
4. 대장정 팀전 2팀 10턴: 파산 발생시켜 즉시 종료 확인.
5. 오목 렌주룰: 쌍삼 자리에 ❌ 표시 + 착수 거부 확인.
6. 화면 꺼짐: 그림 게임 중 화면 잠금 후 복귀 시 캔버스 유지 확인 (초기화되면 알려진 이슈로 기록).
7. 배터리/발열: 폭탄 돌리기 10분 연속 (vibrate 남용 체크).
