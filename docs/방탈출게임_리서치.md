# 방탈출 보드게임 리서치 → 「게르 탈출(er)」 설계 근거

시나리오를 감으로 쓰기 전에, **실제로 잘 팔리는 방탈출 보드게임**을 조사해 "왜 재밌는지 · 얼마나 긴지 · 우리 제약에서 무엇을 훔쳐올지"를 정리한 문서. (조사일: 2026-07-21)

## 1. 시장 리더 (팩트)
| 시리즈 | 규모 | 인원 / 시간 | 검증 방식 | 앱 | 소모성 | 특징 |
|---|---|---|---|---|---|---|
| **EXIT: The Game** (Kosmos) | 압도적 1위, 25+편 | 1~4인 / **1~2시간** | **해독 원판**(decoder disk)에 코드 입력 → 정답 확인 | 선택(타이머/사운드용 Helper 앱) | **카드를 찢고 접음** | 퍼즐이 물리적으로 연쇄. "가장 영리한 퍼즐" 평 |
| **Unlock!** (Space Cowboys) | 2위 | 1~6인 / **60분 하드타이머** | **앱에 코드 입력** → 성공/실패 즉시 피드백. **오답·힌트마다 −3분 페널티** | **필수** | 카드 보존 | 카드 번호를 **더해 조합**("12번+5번 카드"). 앱이 힌트·타이머 담당 |
| **Deckscape** | 라이트 | 1~6인 / ~60분 | 카드 뒤집어 자가 확인 | 없음 | 카드 보존 | 덱만으로. 가볍고 진입장벽 낮음 |

핵심: **한 박스 = 60분 이상, 퍼즐 8~12개**가 업계 표준 볼륨. 우리처럼 하나로 안 끝나고 **여러 편(시리즈)으로 리플레이**를 판다.

## 2. 왜 재밌나 (재미의 정체)
리뷰들이 공통으로 꼽는 4가지:
1. **연쇄(chaining) + 아하 모먼트** — "everything builds on itself." 퍼즐 A를 풀면 나온 조각/코드가 B의 열쇠가 됨. 이 **"아하!"의 연쇄**가 방탈출의 심장. ← *우리 현재 MVP엔 없음(독립 자물쇠 5개)*
2. **퍼즐 타입 다양성** — 논리·시각·수학·암호·물리를 섞어 지루함 방지, 서로 다른 사람이 강점 발휘.
3. **협동 + 몰입** — 한 테이블에 둘러앉아 단서를 물리적으로 만지며 같이 푼다. 테마가 진행을 끌고 감.
4. **검증/피드백 루프** — 원판이 딸깍 맞거나 앱이 "정답!" 하는 **즉각 확인**이 보상. 시간 압박(타이머·페널티)이 긴장 유지.

## 3. 검증된 설계 원칙 (Room Escape Artist · thecodex 압축)
지킬 것:
- **한 퍼즐 = 한 정답**, 모호함 제거. 자리수·형식을 미리 알려줌("4자리 숫자").
- **자가 검증** — 코드/자물쇠로 맞는지 스스로 확인되게.
- **단서↔퍼즐 연결 명확** — 테마·색·라벨·위치로 "이 단서가 저 자물쇠"임이 보이게. 미아 방지.
- **아하 논리가 상식적** — 단서→해답 도약이 현실 논리여야. 제작자 맘대로 X.
- **퍼즐 하나 실행 5분 이내.** 이해한 뒤 노가다 길면 사기 저하.
- **성공/실패 피드백** — 딸깍/흔들림 → 힌트 요청 줄고 만족 상승.
- **난이도 곡선** — 쉬운 걸로 자신감부터, 복잡한 로직은 2~3막에.
- **개인 아하 ↔ 전원 협동 교대** — 모두가 참여하게.
- **3단 자가 힌트** — 넌지 → 핵심 지목 → 단계별 상세.
- 금지: **레드헤링(함정 단서)**, 외부지식 요구, 추측 강요, 대소문자 민감.

## 4. 우리가 어떻게 따라할까 (제약: 오프라인 PWA · 폰 1대 패스앤플레이 · 도트 · 파티)
물리 카드/원판이 없는 대신 **폰이 곧 박스이자 검증기**(= Unlock의 앱 역할). 우리 강점으로 재해석:

1. **연쇄를 넣는다(최우선).** 서브 퍼즐을 풀면 **아이템/단서를 인벤토리에 획득** → 최종 탈출 자물쇠는 그 단서들을 **조합**해야 열림. 지금의 독립 5자물쇠 → **허브형**으로 재구성.
2. **허브 룸 구조.** 게르(방) 한 화면에 관찰 가능한 오브젝트 3~4개 → 아무 순서로 공략(전원 분산 참여) → 각자 코드 획득 → **최종 빗장**은 조합. EXIT 상위편의 병렬 퍼즐을 폰 1대에 맞게 축소한 형태.
3. **도트 관찰 퍼즐 = 우리만의 맛.** 스프라이트 씬에서 개수 세기·별자리 읽기·다른 하나 찾기 → 코드. (에셋 추가 없이 기존 px-sprite 재사용)
4. **볼륨은 파티에 맞춰 축소.** 박스는 60분이지만 캠프파이어 파티엔 길다. **목표 15~25분 · 퍼즐 6~10개**가 스윗스팟. 대신 **시나리오 3~4편**으로 리플레이(EXIT식).
5. **피드백·힌트는 이미 있음** — 흔들림 + 3단 힌트 유지, 해제 "딸깍" 연출 추가.
6. **타이머는 소프트 유지**(파티 친화). 원하면 하드모드(−시간 페널티)는 옵션. — *지금은 YAGNI*
7. 원칙 준수: 자리수 안내, 대소문자 무시(이미 함), 상식적 아하, 레드헤링 금지.

## 5. 결론 · 다음 액션
현 MVP는 루프·검증·힌트·타이머 골격은 맞지만 **방탈출의 심장인 "연쇄+조합"이 빠짐**(독립 자물쇠라 사실상 미니 퀴즈 5개). 다음 작업:
- [ ] er 엔진에 **인벤토리 + 조합 자물쇠** 추가 (허브형 시나리오 지원)
- [ ] 시나리오 1편을 **허브+연쇄 8~10퍼즐**로 재설계 (도트 관찰 퍼즐 2~3개 포함)
- [ ] 이후 시나리오 2~3편 추가로 리플레이 확보
